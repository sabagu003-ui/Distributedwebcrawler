Common Errors & Fixes
Click any error to expand
ERR
ERR_NGROK_334
ngrok tunnel failed to start
›
🔍 Cause A previous ngrok session is still running, blocking the new one.
✅ Fix — Step by Step
1Cell 33 already has ngrok.kill() at the top — this clears the old session.
2Simply re-run Cell 33. The kill command runs first, then a fresh tunnel starts.
3Copy the new URL that appears and paste it into the website.
CONN
Connection Failed
Website can't reach the backend
›
🔍 Cause The ngrok URL is wrong, expired, or Colab stopped running.
✅ Fix — Checklist
1Is the Colab tab still open in your browser? (It disconnects if closed)
2Did Cell 33 finish running? Look for the https://xxxx.ngrok-free.app URL in its output.
3Copy the full URL including https:// and paste it in the website's connect box.
4Click Connect — the dot should turn green.
MAIL
Email Send Failed
Crawl complete alerts not sending
›
🔍 Cause You're using your regular Gmail password instead of a Google App Password.
✅ Fix — Step by Step
1Go to myaccount.google.com → Security → App Passwords
2Generate a new password for "crawler" — you'll get a 16-character code
3Copy it (remove any spaces) and paste into SENDER_PASSWORD in Cell 33
4Re-run Cell 33 and test using the "Send Test Email" button in the Report tab
MOD
ModuleNotFoundError: langdetect
Language detection library missing
›
🔍 Cause The multi-language cells (Part 13) were run before installing the required package.
✅ Fix — Step by Step
1Run Cell 47 first: !pip install langdetect --quiet
2Wait for it to finish (you'll see "Multi-language libraries installed!")
3Then re-run Cell 48 and Cell 49 in order
CRWL
No Results / Empty Crawl
Crawl returns 0 pages or words
›
🔍 Cause URL is malformed, the site blocks scrapers, or the backend isn't connected.
✅ Fix — Checklist
1Make sure the URL starts with https:// — not http or just the domain
2Open the URL in a new browser tab — can you load it manually?
3Try a known-working URL like https://quotes.toscrape.com to test
4Check the Live Log box — it shows exactly which page failed and why
DISC
Colab Disconnects
Runtime timed out or was closed
›
🔍 Cause Google Colab free tier disconnects after ~90 minutes of inactivity.
✅ Fix — Step by Step
1Go back to your Colab notebook and click Reconnect (top right)
2Re-run Cell 33 — a new ngrok URL will be generated
3Paste the new URL into the website and click Connect again
4💡 Tip: Keep your Colab tab active to avoid disconnects during long crawls
